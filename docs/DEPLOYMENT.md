# Deployment Guide

## Live Deployment

| Service | Platform | URL |
|---|---|---|
| Frontend | Vercel | https://job-board-eight-henna.vercel.app |
| Backend API | Render | https://job-board-gfnq.onrender.com |
| Database | Render PostgreSQL | Auto-linked via DATABASE_URL |

---

## Frontend — Vercel

1. Push repo to GitHub
2. Go to [vercel.com](https://vercel.com) → **Add New Project** → import repo
3. Set **Root Directory** to `frontend`
4. Add environment variable:
   - `NEXT_PUBLIC_API_URL` = `https://job-board-gfnq.onrender.com/api/v1`
5. Click **Deploy**

Vercel auto-deploys on every push to `main` via GitHub integration.

---

## Backend — Render

1. Go to [render.com](https://render.com) → **New Web Service** → import repo
2. Set **Root Directory** to `backend`
3. Set **Build Command**:
```
pip install -r requirements/prod.txt && python manage.py collectstatic --noinput && python manage.py migrate --noinput
```
4. Set **Start Command**:
```
gunicorn config.wsgi:application --bind 0.0.0.0:$PORT
```
5. Add environment variables:

| Key | Value |
|---|---|
| `DJANGO_SETTINGS_MODULE` | `config.settings.production` |
| `DJANGO_SECRET_KEY` | Auto-generated by Render |
| `DATABASE_URL` | Auto-linked from PostgreSQL |
| `ALLOWED_HOSTS` | `job-board-gfnq.onrender.com` |
| `CORS_ALLOWED_ORIGINS` | `https://job-board-eight-henna.vercel.app` |
| `CLOUDINARY_CLOUD_NAME` | From Cloudinary dashboard |
| `CLOUDINARY_API_KEY` | From Cloudinary dashboard |
| `CLOUDINARY_API_SECRET` | From Cloudinary dashboard |

---

## Database — Render PostgreSQL

1. Go to Render → **New PostgreSQL**
2. Set region to **Virginia (US East)** — same as web service
3. Select **Free** plan
4. Link to web service via `DATABASE_URL`

---

## CI/CD Pipeline — GitHub Actions

GitHub Actions runs automatically on every push to `main`:

- Backend: Django system checks + migration validation
- Frontend: ESLint + production build (`next build`)

Vercel and Render auto-deploy on successful push.
